# weather-cli

A small command-line tool that fetches the current weather for any city from the
[OpenWeatherMap API](https://openweathermap.org/api) and prints a colourful report —
complete with ASCII-art weather icons — to your terminal.

It can also run as a tiny Express web server that exposes the same data over HTTP.

```
    \   /
     .-.
  --(   )--
     '-'
    /   \

Weather Report for Dubai
Condition: clear sky
Temperature: 34°C
Feels like: 38°C
Humidity: 55%
```

## Features

- Look up current weather by city name (`--city`)
- Temperature, "feels like", humidity and conditions in metric units
- Coloured output with condition-based ASCII icons (`chalk`)
- Optional HTTP mode: `GET /weather?city=<name>` returns JSON

## Requirements

- [Node.js](https://nodejs.org/) 18+
- A free OpenWeatherMap API key — sign up at https://openweathermap.org/api

## Setup

1. **Clone and install dependencies**

   ```bash
   git clone <repo-url>
   cd weather-cli
   npm install
   ```

2. **Add your API key**

   Create a `.env` file in the project root:

   ```bash
   API_KEY=your_openweathermap_api_key_here
   PORT=3000   # optional, only used in web-server mode
   ```

   `.env` is git-ignored, so your key stays out of version control.

## Usage

### Quick start (no build, uses ts-node)

```bash
npm start -- --city "London"
# or the short alias
npm start -- -c "London"
```

> The `--` passes the flags through npm to the script.

City defaults to `Dubai` if the option is omitted in code, but `--city` is
required by the CLI parser, so always pass one.

### Built version

Compile to `dist/` first, then run:

```bash
npm run build
npm run weather -- "Tokyo"     # runs: node dist/index.js --city Tokyo
```

### Watch mode (development)

Auto-restarts on file changes:

```bash
npm run dev -- --city "Paris"
```

### Help

```bash
npm start -- --help
```

## Web-server mode

When `index.ts` is **imported** rather than run directly, it starts an Express
server instead of executing the CLI. To run it as a server, create a small entry
that imports the module, or adapt the entrypoint. Once running it exposes:

```
GET /weather?city=London
```

Response:

```json
{
  "city": "London",
  "weather": "broken clouds",
  "temperature": 14.2,
  "feelsLike": 13.5,
  "humidity": 72,
  "formattedOutput": "...coloured terminal string..."
}
```

The server listens on `PORT` (default `3000`).

### Try it live

A hosted instance is running on Render. Try it without setting anything up —
this fetches Dubai's weather and prints the coloured terminal report:

```bash
curl -s "https://weather-cli-mrlm.onrender.com/weather?city=Dubai" | jq -r .formattedOutput
```

Swap `Dubai` for any city. Requires [`jq`](https://stedolan.github.io/jq/) to
extract and render the `formattedOutput` field.

## npm scripts

| Script          | What it does                                              |
| --------------- | -------------------------------------------------------- |
| `npm start`     | Run the CLI with `ts-node` (no build step)               |
| `npm run dev`   | Run with `nodemon` + `ts-node`, restarting on changes    |
| `npm run build` | Type-check with `tsc`, then bundle with `bun`            |
| `npm run weather` | Run the compiled CLI from `dist/` (`build` first)      |

## Project structure

```
src/
  index.ts         CLI + Express server, API calls, output formatting
  weatherIcons.ts  Map of weather conditions → ASCII-art icons
dist/              Compiled output (generated by `npm run build`)
.env               Your API key (not committed)
```

## Troubleshooting

- **`401 Unauthorized`** — your `API_KEY` is missing or invalid. New keys can
  take a few minutes to activate.
- **`404 Not Found` / "city not found"** — check the spelling, or qualify the
  city, e.g. `--city "London,GB"`.
- **`dist/index.js not found`** — run `npm run build` before `npm run weather`.
